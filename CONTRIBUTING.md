# Contributing

感谢你愿意参与这个项目。

## 开始之前

- 先阅读 [README.md](./README.md) 了解项目范围
- 先阅读 [Product-Spec.md](./Product-Spec.md) 确认需求边界
- 涉及功能新增或行为变更时，先更新需求文档，再改代码

## 提交建议

- Bug 修复：请说明复现步骤、预期结果和实际结果
- 功能建议：请说明使用场景、问题背景和预期交互
- UI 调整：请尽量附上设计说明、草图或截图

## 开发流程

1. Fork 仓库并创建功能分支
2. 在本地完成修改与自测
3. 确认没有把构建产物、截图和本地配置文件提交进来
4. 更新必要文档
5. 提交 Pull Request

## Pull Request 要求

- 标题清晰，能说明改动目的
- 描述中说明背景、范围和验证方式
- 如果改动影响 UI，请补充截图或录屏
- 如果改动影响需求，请同步更新 `Product-Spec.md`

## 提交前检查

- 可以正常 Gradle Sync
- `.\gradlew.bat assembleDebug` 可以通过
- 没有提交 `local.properties`、`build/`、`.gradle/` 等本地文件

## 沟通约定

- 优先使用清晰、可复现、可验证的描述
- 对外部设备、固件和接口的假设请写明
- 不确定的能力边界不要硬猜，直接在 Issue 或 PR 中说明
